EM Tech Internal Handbook

EM Tech is a software consulting company that focuses on building scalable cloud-native applications.

Technology Stack:
- Java 21
- Spring Boot
- React
- PostgreSQL
- Kubernetes
- Docker

Engineering Principles:

1. Clean Code
Engineers should write readable and maintainable code.

2. API First Development
All backend services must expose REST APIs documented using OpenAPI.

3. Observability
All services should include logging, metrics, and tracing.

4. Security
All APIs must implement OAuth2 authentication.

Development Workflow:

- Feature branches must be created for every task.
- Pull requests must be reviewed by at least one engineer.
- CI pipeline must pass before merging.

Testing Guidelines:

- Unit tests should cover business logic.
- Integration tests should validate database interactions.
- Test coverage must remain above 80%.
